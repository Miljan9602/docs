---
title: "Get Daily Analytics"
api: "GET /api/v1/analytics/daily"
---

Get a daily breakdown across a date range, with one data point per UTC day. Each day reports total volume, trade count, and fees. Days with no successful trades are **still included** with all-zero values so consumers can render a full-range chart without having to reconcile missing buckets. The response also includes a `totals` object summing the whole range.

Data is read from the pre-aggregated `analytics_daily` rollup table (populated hourly by the `analytics-rollup` cron). See [Get Hourly Analytics](/frontend/api/get-analytics-hourly) for per-hour granularity within a single day.

## Query Parameters

<ParamField query="from" type="string">
  Start date (inclusive) in ISO 8601 format `YYYY-MM-DD`. If omitted, defaults to the earliest day with data for the given filter (i.e. all-time).
</ParamField>

<ParamField query="to" type="string">
  End date (inclusive) in ISO 8601 format `YYYY-MM-DD`. If omitted, defaults to today (UTC). Must be greater than or equal to `from`.
</ParamField>

<ParamField query="chain_id" type="string">
  Filter by chain ID. If omitted, returns aggregate stats across all chains.
</ParamField>

## Response

<ResponseField name="status" type="string">
  Response status: `"ok"` on success.
</ResponseField>

<ResponseField name="from" type="string">
  The effective start date of the returned range (echo of `from`, or the earliest day with data if `from` was omitted).
</ResponseField>

<ResponseField name="to" type="string">
  The effective end date of the returned range (echo of `to`, or today UTC if `to` was omitted).
</ResponseField>

<ResponseField name="totals" type="object">
  Summary totals across every day in the returned range.

  <Expandable title="Totals">
    <ResponseField name="volume" type="number">
      Total USD volume for the whole range.
    </ResponseField>

    <ResponseField name="trade_count" type="number">
      Total number of successful trades across the range.
    </ResponseField>

    <ResponseField name="fees" type="number">
      Total USD protocol fees collected across the range.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="data_points" type="array">
  One entry per UTC day from `from` to `to` (inclusive), sorted chronologically. Days with no successful trades are present with zero values.

  <Expandable title="Data Point">
    <ResponseField name="timestamp" type="number">
      Unix timestamp in seconds for midnight UTC of the day.
    </ResponseField>

    <ResponseField name="volume" type="number">
      Total USD volume for the day. `0` if no trades.
    </ResponseField>

    <ResponseField name="trade_count" type="number">
      Number of successful trades for the day. `0` if none.
    </ResponseField>

    <ResponseField name="fees" type="number">
      Total USD protocol fees collected for the day. `0` if no trades.
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
# All-time, Sei mainnet
curl -X GET "https://rfq.saphyre.xyz/api/v1/analytics/daily?chain_id=1329"

# Explicit range
curl -X GET "https://rfq.saphyre.xyz/api/v1/analytics/daily?from=2026-04-01&to=2026-04-22&chain_id=1329"
```

```javascript JavaScript
const params = new URLSearchParams({
  from: "2026-04-01",
  to: "2026-04-22",
  chain_id: "1329",
});

const response = await fetch(
  `https://rfq.saphyre.xyz/api/v1/analytics/daily?${params}`
);
const data = await response.json();

console.log(`Range: ${data.from} → ${data.to}`);
console.log(`Totals: $${data.totals.volume} / ${data.totals.trade_count} trades / $${data.totals.fees} fees`);
for (const point of data.data_points) {
  const day = new Date(point.timestamp * 1000).toISOString().slice(0, 10);
  console.log(`${day}: $${point.volume} / ${point.trade_count} trades / $${point.fees} fees`);
}
```

```python Python
import requests
from datetime import datetime, timezone

response = requests.get(
    "https://rfq.saphyre.xyz/api/v1/analytics/daily",
    params={"from": "2026-04-01", "to": "2026-04-22", "chain_id": "1329"},
)
data = response.json()

print(f"Range: {data['from']} -> {data['to']}")
print(f"Totals: ${data['totals']['volume']} / {data['totals']['trade_count']} trades / ${data['totals']['fees']} fees")
for point in data["data_points"]:
    day = datetime.fromtimestamp(point["timestamp"], tz=timezone.utc).date()
    print(f"{day}: ${point['volume']} / {point['trade_count']} trades / ${point['fees']} fees")
```
</RequestExample>

## Notes

- Public endpoint; no authentication required.
- Daily buckets are keyed by midnight UTC (`00:00:00Z`) of each day.
- **Always zero-filled**: every day from `from` to `to` (inclusive) is present in `data_points`, even if it had no trades. This makes the response chart-ready.
- Volume and fee values are in **USD** (summed from `analytics_daily.volume_usd` and `analytics_daily.fees_usd`).
- Cache TTL: 60 seconds if the range includes today (data still mutating), 1 hour otherwise.
- Unbounded ranges: if `from` is omitted, the server uses the earliest day with data for that chain filter; this auto-scopes the response to actual observed history.
- For hourly granularity within a single day, use [`/api/v1/analytics/hourly`](/frontend/api/get-analytics-hourly).
